Manual

MITgcmTools.jl provides a suite of tools for analyzing MITgcm results, compiling the model, modifying its inputs, running simulations, etc all from within julia. Much of the functionalities are documented via the Examples section, as highlighted hereafter.

Read / Write MITgcm Files

Let's assume that user has run an MITgcm configuration in standard fashion, and wants to examine the output in Julia. This is one of the common use case for MITgcmTools.jl.

The tutorial_global_oce_biogeo, which comes with MITgcm, provides a representative example. With MITgcmTool.jl you can just run it in a notebook (MITgcm_tutorial_global_oce_biogeo.jl).

Note

Alternatively, the MITgcm documentation explains how to complile and run tutorials at the linux command line.

Once the model has run, it's output is found in the run/ folder (see notebook). The scan_rundir function can provide a summary information about files in the run/ folder.

rundir=joinpath(pth,"run")
sc=scan_rundir(rundir)
sc.completed

The result will indicate whether the model run has complete normally based upon the Standard Output file (output.txt or STDOUT.000).

The scan_rundir function checks whether the model output is format binary (sc.params_files.use_mdsio) or NetCDF (sc.params_files.use_mnc). It also identifies the type of grid that was used (sc.params_grid).

With this information, we are ready to read model output. The various files that MITgcm can generate are covered below.

  • Scan Run Folder
  • MDS Files (binary)
  • MNC Files (netcdf)
  • Grid Files (binary or netcdf)
  • Standard Output (text)
  • Input Files (text)
  • Other Files

Grid variables are often needed for analysis. The grid output can be read from file using either GridLoad_mdsio or GridLoad_mnc. This will return Γ.XC, Γ.YC, etc formated using MeshArrays.jl. See also GridLoad_native.

Note

MITgcm_scan_output.jl does this in bulk for all configurations in MITgcm/verification and further displays each grid (this page).

MITgcm Configurations

The verification_experiments function lists standard model configurations found in the joinpath(MITgcm_path[1],"verification") folder.

Each model configuration has a subfolder where MITgcm can be compiled and run.

In MITgcmTools.jl, a model configuration is represented as a MITgcm_config. This data structure allows you take advantage of the ClimateModels.jl interface for example.

The setup function can thus be used to prepare a temporary run directory for the MITgcm_config; then build to compile the model, and MITgcm_launch to run it.

Note

For more on these aspects, see Examples, More On Configurations, and ClimateModels Interface.
